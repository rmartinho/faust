FAUST

Faction and Unit Stat Tables

What is FAUST?

FAUST is a website generator for visualizing unit stats for Rome: Total War mods. It supports mods for the original Rome: Total War as well as Total War: Rome Remastered, and there is also some support for Medieval II: Total War mods.

If you want to see what that looks like, take a look at one of these previews.

How do I get FAUST?

You can download the latest version of FAUST from the releases page. FAUST works on multiple operating systems, so choose the one that matches yours and download the corresponding zip file. Inside you will find an executable file, a README with instructions, and a file describing what’s changed from previous versions.

How do I use FAUST?

Once you have downloaded the latest release for your operating system, you need to prepare a manifest. The manifest is a file that describes how you want to process your mod. Inside the mod folder, create a folder named faust. This is, next to the data folder, not inside it. All of the FAUST-specific files will go in that folder.

Inside the faust folder, create a text file named faust.yml. This will be your manifest file. This file will be written in YAML syntax. Don’t worry if you’re not familiar with that; it should be quite straighforward once you see examples. Here’s an example of a simple manifest. 

id: rome_partial_war
name: "Rome: Partial War"
game: original

A manifest like this first declares an ID for the mod, which will be used internally and also in URL. Then it declares the display name of the mod. This will be shown in page content and page titles. And lastly, it defines which game the mod is for, so FAUST knows how to process the mod folder. The three possibilities are original, remastered, and medieval2.

There are more options available but they are optional and you may or may not need to use them. These three are all that is required. If you want to see what other options are available, you can scroll down to that section.

Once you have a manifest file, you still need one more thing: a banner image to represent your mod. This is just an image file named banner.png next to the manifest file. This image can be whatever you want, but you should know that ir will be displayed as a 512x256 image.

Now that you have everything ready, it’s time to launch FAUST. FAUST is a command-line application, but the Windows version has some niceties that make it unnecessary to mess with the command-line directly.

Easy mode

If you’re on Windows, you can just double-click the executable file and it will show a dialog for you to select your manifest file. Just navigate to the foldee where you created it earlier and select it.

Advanced mode

If you are not on Windows, or you prefer to use the command-line directly, you can launch the faust executable passing the path to manifest as an argument. Further down this document you can find more details on the command-line options that are available.

Once you launched FAUST with your manifest file, you will see some progress displayed. If everything goes well, FAUST will read in the files from your mod folder, build a catalog of all the factions and units, and then write out a website that displays all of that information.

If you launched FAUST in easy mode, or if you used the -s command-line option, FAUST will then start a local web server so that you can browse the website it generated. The address will be copied to your clipboard, so you can just paste it in your browser’s address bar.

What are the advanced manifest options?

Here is a list of all of the options supported in the manifest. You can also have a look at a comprehensive example.

id

Required. The mod ID. This is used internally in scripts and in the URLs.

Example
id: rome
name

Required. The mod’s display name. This is used in the generated website as the page title.

Example
name: "Rome: Total War"
game

Optional. Which game this mod is for. This is used to let FAUST know how to process the mod files.

Value Meaning
original Rome: Total War
remastered Total War: Rome Remastered (default)
medieval2 Medieval II: Total War
Example
game: original
campaign

Optional The campaign folder to process. Currently FAUST only supports cataloguing one campaign. This option lets you specify the name of the campaign folder that should be processed. By default, the following folder names are tried in order:

  1. imperial_campaign
  2. barbarian_invasian
  3. alexander
Note: This option is likely to be changed or removed in a future update.
Example
campaign: sons_of_mars
banner

Optional. The banner image file. This can be an absolute path or relative to the manifest folder. By default FAUST assumes the banner is a file named banner.png in the manifest folder.

Example
banner: rome.tga
dir

Optional. The mod folder. This can be an absolute path or relative to the manifest folder. This is useful if you want to keep the manifest outside of the mod folder. By default FAUST assumes the manifest folder is at the root of the mod folder.

Example
dir: ../mod
exclude

Optional. Which factions to exclude when generating the site. Sometimes mods need to keep factions that are not used, e.g. to avoid some crashes. In order to exclude those factions from the catalog, you can list those unused factions with this option in the manifest. By default all factions that have units are included.

Example
exclude:
  - gauls
  - britons
  - germans
aliases

Optional. Faction ID aliases. Sometimes mods reuse the base game faction IDs but the factions are changed to something completely different. Since the faction IDs are used in URLs, it might be confusing to see those unrelated faction IDs. In order to improve this, you can specify aliases that will be used the URLs instead of the internal faction IDs. By default, no aliases are used.

Example
aliases:
  britannia: britons
  germania: germans
eras

Optional. Many mods have units that become available only after certain events occur. The most notable example is the Marian reforms in the base game. The mod files don't have an explicit concept of eras like this, but you can define them yourself with this option. By default, no eras are defined.

Example
eras:
  pre_marian:
    name: "Before the Marian reforms"
    major_event:
      marian_reforms: false
  post_marian:
    name: "After the Marian reforms"
    major_event:
      marian_reforms: true

Additionally, you should create a folder named eras in the manifest folder, and create two image files for each era in it: one to use when the era is selected, and one to use when the era is not selected. For the example above, the files would be named as follows.

  1. pre_marian-on.png
  2. pre_marian-off.png
  3. post_marian-on.png
  4. post_marian-off.png
unit_info_images

Optional. Whether to use images from ui/unit_info instead of ui/unit. Use this option if the mod has small unit card images (e.g. most mods for the original game) that end up looking blurry at the scale used in the site. This will instead use the unit info images which are larger. By default, FAUST will use unit card images for remastered mods, and unit info images for others.

Example
unit_info_images: true
speed

Optional. Whether to estimate speeds. If `true`, FAUST will perform a speed estimation based on the base game skeleton animations and display it. If the estimation is off, you can use the `speeds` option to directly provide each skeleton’s speed. By default, no speed estimates are done.

Example
speed: true
speeds

Optional. Speed mapping. Currently the unit move speeds are an estimation based on the base game skeleton animations. Each skeleton is internally mapped to a hardcoded speed. This option exists to override this, in case there are different skeletons. If specified, `speed` is set to true.

Example
speeds:
  fs_dagger: 30
  fs_semi_fast_dagger: 32
pools

Optional. Mercenary pool information. The game doesn't have any text strings for the mercenary pools; this option lets you specify names for the mercenary pools to be displayed in the mercenaries page. This is just a list of names for the pools in the same order as in descr_mercenaries.txt. By default, the internal pool IDs are used.

Example
pools:
  - "Britannia"
  - "Germania"
  - "Eastern Europe"
  - "Steppes"
  - "Central Europe"
  - "Gaul"
aors

Optional. Area of recruitment information. The game doesn't have any text strings for areas of recruitment; like pools above, this option lets you specify names for the areas of recruitment to be displayed in the regional units pages.

This is also a list of names in order. However, because areas of recruitment have no explicit realization in the game (they're defined with the hidden resources system), this order is arbitrary, but it's fixed for each mod. You can generate the site once and look in images/<mod-id>/aors to see what the correct order is. By default, AORs have no associated names.

Example
aors:
  - "Spartans"
  - "Latins"
What about the command-line options? 
Usage: faust [OPTIONS] [MANIFEST]

Arguments:
  [MANIFEST]  the manifest file

Options:
  -o, --out-dir <OUT_DIR>                where to output the site
  -b, --base-game-path <BASE_GAME_PATH>  base game path (for fallbacks)
  -s, --serve                            serve the site after generation
  -h, --help                             Print help
  -V, --version                          Print version
